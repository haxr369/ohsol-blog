## 🧭 전반 방향

현재 환경은 **Spring Boot, Kafka, CI/CD 개선 등의 신기술 도입이 제한된 운영 환경**임.  
따라서 회사 내에서는 **운영 안정화·성능 개선·협업 능력 강화**를 중심으로 성장하고,  
개인적으로는 **차세대 기술 스택(Spring, Kafka, PostgreSQL, AWS 등)** 을 학습하여  
미래 시스템 전환 또는 개인 프로젝트를 통한 성장 기반을 확보한다.

---

## 🏢 회사 내 성장 목표 (운영 환경 중심)

### 🎯 핵심 방향
- 자산평가 서비스 운영 효율화 및 신뢰성 강화  
- 대용량 배치/실시간 평가 구조 최적화  
- 현업과의 협업 및 문제해결력 고도화

### ✅ 주요 목표
| 구분 | 목표 | 세부 내용 |
| --- | --- | --- |
| **성능 최적화** | 대용량 배치 및 실시간 조회 성능 개선 | Hash Join, KEEP, Partition 활용 등 Oracle 기반 성능 개선 |
| **운영 안정화** | 장애 대응력 강화 및 이력 관리 체계화 | 점검 이유 → 원인 → 해결 과정을 문서화 |
| **협업 능력 강화** | 현업/타부서 요청 대응 프로세스 표준화 | 서비스 구조를 명확히 설명하고, 요청별 SLA 관리 |
| **업무 도메인 이해도 강화** | 자산평가 서비스 전체 구조 문서화 | 평가 로직, 시장별 데이터 흐름, EAI·BP 호출 경로 시각화 |

### 💡 실현 예시
- 주기적 점검 업무 SQL 자동화 템플릿 구축  
- 배치 성능 개선을 위한 실행계획 분석 체계화  
- 자산평가 구조도를 Notion/Wiki 형태로 정리해 신규 인력 온보딩 지원
- 달레 리트코드 스터디에 참여해서 알고리즘 문제 해결 능력 강화

## 💻 개인 프로젝트 및 학습 목표 (기술 성장 중심)

### 🎯 핵심 방향
- 새로운 기술 스택을 실습형으로 학습  
- 실제 자산평가 유사 시나리오를 개인 프로젝트로 구현  
- 기술 문서화 및 코드 품질 향상 중심 학습

### ✅ 주요 목표
| 구분 | 목표 | 세부 내용 |
| --- | --- | --- |
| **Spring Boot 학습** | 자산평가 서비스 미니 백엔드 구축 | REST API, 트랜잭션, 예외 처리 중심 |
| **Kafka 학습** | 실시간 평가 이벤트 스트림 구축 | 평가 결과를 Kafka 토픽으로 전달하고 모니터링 |
| **PostgreSQL 학습** | 대체 Oracle 구조 모델링 및 성능 비교 | Partition, Index 전략 비교 실험 |
| **AWS 학습** | 클라우드 기반 배포 환경 구축 | EC2, RDS, S3, CloudWatch 활용 |
| **CI/CD 학습** | GitHub Actions 중심 간단한 파이프라인 구성 | 테스트 자동화 및 배포 시뮬레이션 |

### 💡 개인 프로젝트 예시
#### 프로젝트명: “AssetFlow — 실시간 자산평가 시스템”
- Stack: Spring Boot + Kafka + PostgreSQL + Redis + AWS  
- 기능:
  - 고객별 종목 자산평가 API 제공
  - Kafka 기반 실시간 평가 결과 스트림 전송
  - PostgreSQL Partition으로 대용량 평가 데이터 관리
- 학습 포인트:
  - Oracle KEEP 문법 → PostgreSQL Window Function으로 대체
  - EAI/WAS 호출 구조를 Kafka 기반 이벤트 전송으로 재현
